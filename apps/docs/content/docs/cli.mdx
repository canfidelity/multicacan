The Multicacan CLI mirrors almost everything the Web UI can do (create [issues](/issues), assign [agents](/agents), start the [daemon](/daemon-runtimes), and more). This page lists every top-level command with a one-line description. For the full set of flags and examples, run `multicacan <command> --help`.

## Getting authenticated

Run this the first time you use the CLI to obtain a **personal access token (PAT)**:

```bash
multicacan login
```

Your browser opens automatically. After you approve in the web app, the CLI saves the PAT (prefixed with `mul_`) to `~/.multicacan/config.json`. Every subsequent command authenticates with that PAT.

<Callout type="tip">
For CI or headless environments, skip the browser flow: create a PAT in the web app under **Settings → Personal Access Tokens**, then run `multicacan login --token <mul_...>` to supply it directly.
</Callout>

For the difference between token types, see [Authentication and tokens](/auth-tokens).

## Auth and setup

| Command | Purpose |
|---|---|
| `multicacan login` | Log in and save a PAT |
| `multicacan auth status` | Show current login status, user, and workspace |
| `multicacan auth logout` | Clear the local PAT |
| `multicacan setup cloud` | One-shot setup for Multicacan Cloud (login + install daemon) |
| `multicacan setup self-host` | One-shot setup for a self-hosted backend |

## Workspaces and members

| Command | Purpose |
|---|---|
| `multicacan workspace list` | List every workspace you can access |
| `multicacan workspace get <slug>` | Show details for one workspace |
| `multicacan workspace members` | List members of the current workspace |

## Issues and projects

| Command | Purpose |
|---|---|
| `multicacan issue list` | List issues |
| `multicacan issue get <id>` | Show a single issue |
| `multicacan issue create --title "..."` | Create a new issue |
| `multicacan issue update <id> ...` | Update an issue (status, priority, assignee, etc.) |
| `multicacan issue assign <id> --agent <slug>` | Assign to an agent (triggers a task immediately) |
| `multicacan issue status <id> --set <status>` | Shortcut to change status |
| `multicacan issue search <query>` | Keyword search |
| `multicacan issue runs <id>` | Show agent runs on an issue |
| `multicacan issue rerun <id>` | Rerun the most recent agent task |
| `multicacan issue comment <id> ...` | Nested: view / post comments |
| `multicacan issue subscriber <id> ...` | Nested: subscribe / unsubscribe |
| `multicacan project list/get/create/update/delete/status` | Project CRUD |

## Agents and skills

| Command | Purpose |
|---|---|
| `multicacan agent list` | List the workspace's agents |
| `multicacan agent get <slug>` | Show an agent's configuration |
| `multicacan agent create ...` | Create an agent |
| `multicacan agent update <slug> ...` | Update an agent |
| `multicacan agent archive <slug>` | Archive |
| `multicacan agent restore <slug>` | Restore an archived agent |
| `multicacan agent tasks <slug>` | Show an agent's task history |
| `multicacan agent skills ...` | Nested: attach / detach skills |
| `multicacan skill list/get/create/update/delete` | Skill CRUD |
| `multicacan skill import ...` | Import a skill from GitHub, ClawHub, or the local machine |
| `multicacan skill files ...` | Nested: manage a skill's files |

## Autopilots

| Command | Purpose |
|---|---|
| `multicacan autopilot list` | List every autopilot in the workspace |
| `multicacan autopilot get <id>` | Show a single autopilot |
| `multicacan autopilot create ...` | Create an autopilot |
| `multicacan autopilot update <id> ...` | Update |
| `multicacan autopilot delete <id>` | Delete |
| `multicacan autopilot runs <id>` | Show run history |
| `multicacan autopilot trigger <id>` | Trigger a run manually |

## Daemon and runtimes

| Command | Purpose |
|---|---|
| `multicacan daemon start` | Start the daemon (background by default; add `--foreground` to run in the foreground) |
| `multicacan daemon stop` | Stop the daemon |
| `multicacan daemon restart` | Restart the daemon |
| `multicacan daemon status` | Check whether the daemon is online and its concurrency |
| `multicacan daemon logs` | View daemon logs |
| `multicacan runtime list` | List runtimes in the current workspace |
| `multicacan runtime usage` | Show resource usage |
| `multicacan runtime activity` | Recent activity log |
| `multicacan runtime ping <id>` | Ping a runtime to check it's online |
| `multicacan runtime update <id> ...` | Update a runtime's configuration |

## Miscellaneous

| Command | Purpose |
|---|---|
| `multicacan repo checkout <url>` | Clone a repo locally for agents to use |
| `multicacan config` | View or edit local CLI configuration |
| `multicacan version` | Print the CLI version |
| `multicacan update` | Upgrade the CLI to the latest release |
| `multicacan attachment download <id>` | Download an attachment from an issue or comment |

## Getting full flags

Every command supports `--help`:

```bash
multicacan issue create --help
multicacan agent update --help
```

v2 will ship a dedicated detailed reference page for each command.
